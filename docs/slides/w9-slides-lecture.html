What Is R Shiny?

The Short Answer

  • Shiny is an R framework for building interactive web apps.
  • Users control inputs such as dropdowns, buttons and sliders.
  • R responds by updating charts, tables or text.

Why Analysts Use It

  • Users can explore data without editing code.
  • One app can replace many repeated charts.
  • You can share analysis as a tool, not just a report.

The Mental Model

user changes input
  -> server code reacts
  -> output updates

The user does not see the R code. They see the app.

Static Chart vs Shiny App

Static chart:
  "Here is Rui Hachimura."

Shiny app:
  "Choose any player, and the chart updates."

Anatomy of a Shiny App

One File: app/app.R

A basic Shiny app has four parts:

app/app.R
library(shiny)

# 1. Load data

# 2. Define UI

# 3. Define server

# 4. Launch app
shinyApp(ui = ui, server = server)

The Two Core Objects

ui
  what the user sees

server
  what R computes

Keep that split clear.

UI

What the UI Does

The UI defines the page.

It contains:

  • inputs, such as selectInput();
  • output placeholders, such as plotOutput();
  • layout, such as fluidPage() or sidebarLayout().

Minimal UI

app/app.R
ui <- fluidPage(
  plotOutput("zone_bars")
)

This says: put a plot on the page and call it zone_bars.

UI with a Dropdown

app/app.R
player_choices <- sort(unique(shot_zones$player_name))

ui <- fluidPage(
  selectInput(
    inputId = "player",
    label = "Player",
    choices = player_choices,
    selected = "Rui Hachimura"
  ),
  plotOutput("zone_bars")
)

The dropdown value is stored as input$player.

UI with Layout

app/app.R
ui <- fluidPage(
  titlePanel("Lakers Evolution Dashboard"),
  sidebarLayout(
    sidebarPanel(
      selectInput("player", "Choose a player", player_choices)
    ),
    mainPanel(
      plotOutput("zone_bars", height = "500px")
    )
  )
)

Inputs usually go in the sidebar. Outputs usually go in the main panel.

Server

What the Server Does

The server fills the UI placeholders.

UI:
  plotOutput("zone_bars")

Server:
  output$zone_bars <- renderPlot(...)

The names must match.

Minimal Server

app/app.R
server <- function(input, output, session) {
  output$zone_bars <- renderPlot({
    shot_zones |>
      filter(player_name == "Rui Hachimura") |>
      ggplot(aes(x = shot_zone_basic, y = share, fill = era)) +
      geom_col(position = position_dodge(width = 0.8), width = 0.7)
  })
}

This proves the plot can live inside Shiny.

Reactive Server

app/app.R
server <- function(input, output, session) {
  output$zone_bars <- renderPlot({
    shot_zones |>
      filter(player_name == input$player) |>
      ggplot(aes(x = shot_zone_basic, y = share, fill = era)) +
      geom_col(position = position_dodge(width = 0.8), width = 0.7)
  })
}

Now the chart changes when the dropdown changes.

Reactivity

The Key Idea

You do not write:

when the dropdown changes, redraw the chart

Shiny handles that.

You write code that uses input$player, and Shiny watches it.

Dependency

selectInput("player", ...)
        |
        v
input$player
        |
        v
renderPlot({ ... })
        |
        v
plotOutput("zone_bars")

If input$player changes, renderPlot() reruns.

Names Must Match

selectInput("player", ...)
input$player

plotOutput("zone_bars")
output$zone_bars <- renderPlot(...)

Most beginner Shiny bugs are name mismatches.

Build It in Stages

Stage 1: Minimal App

  • Load shot_zones.rds.
  • Show one chart.
  • Hard-code one player.

Goal: prove the app runs.

Stage 2: Add an Input

  • Create player_choices.
  • Add selectInput().
  • Replace "Rui Hachimura" with input$player.

Goal: make the chart reactive.

Stage 3: Add Layout

  • Add titlePanel().
  • Use sidebarLayout().
  • Put controls in sidebarPanel().
  • Put the chart in mainPanel().

Goal: make the app easier to use.

Stage 4: Optional Plotly Output

The same UI/server pairing applies to Plotly.

plotOutput()    -> renderPlot()
plotlyOutput()  -> renderPlotly()

The Shiny idea is the same: output placeholder in the UI, render function in the server.

Running the App

Start the App

From the project root:

R code
shiny::runApp("app")

This opens the local app in the RStudio viewer or browser.
